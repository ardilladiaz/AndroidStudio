<h1>Desarrollo de Interfaz Alke Wallet</h1>
<p><strong>Evaluación Módulo #4</strong> | Desarrollo de Aplicaciones Móviles</p>

<h2>1. Introducción</h2>
<p>El presente informe documenta el proceso de desarrollo de la interfaz de usuario (UI) para la aplicación financiera <strong>Alke Wallet</strong>, correspondiente a la evaluación del Módulo 4[cite: 1, 4]. El objetivo principal consistió en la creación de 9 pantallas independientes utilizando <strong>Android Studio</strong> y lenguaje <strong>XML</strong> para el diseño de los layouts. El proyecto se enfocó en trasladar los recursos de Figma a un entorno de desarrollo funcional, priorizando la jerarquía visual y la coherencia estética requerida para una empresa del sector financiero.</p>

<h2>2. Desarrollo Técnico y Metodología</h2>
<h3>2.1. Fuentes de Consulta y Aprendizaje</h3>
<p>Para la ejecución técnica, se adoptó una ruta de aprendizaje basada en:</p>
<ul>
<li>
<strong>Capacitación Especializada:</strong> Uso de tutoriales de <em>MaxCodeDev</em> para el dominio de Android Studio, específicamente en la implementación de layouts XML y la estructuración de archivos en Java.
</li>
<li>
<strong>Recursos de Diseño:</strong> Integración de archivos (assets), iconos y paleta de colores extraídos directamente del archivo Figma referenciado en la consigna[cite: 336, 349, 350], además de algunos de uso libre de <em>TheNounProject</em>
</li>
</ul>
<h3>2.2. Criterio de Diseño y Optimizaciones (Perfil Diseñadora)</h3>
<p>Dada mi formación previa en diseño, realicé ajustes críticos sobre el prototipo original para mejorar la usabilidad y la estética:</p>
<ul>
<li>
<strong>Jerarquía de Texto y Accesibilidad:</strong> Una de las decisiones principales fue aumentar el tamaño de los textos en los botones[cite: 185, 200]. En el diseño original, el puntaje resultaba insuficiente para una visualización cómoda y una lectura rápida; esta modificación garantiza que el usuario identifique claramente las acciones principales de la billetera.</li>
<li>
<strong>Ajustes de Composición:</strong> Se refinó la disposición de elementos para lograr un equilibrio visual superior al propuesto inicialmente, buscando una interfaz más limpia y profesional.
</li>
<li>
<strong>Tipografía de Sistema:</strong> Siguiendo un enfoque ágil, se utilizaron las fuentes por defecto de Android debido a la complejidad técnica de importar familias externas en esta fase. Se considera una decisión estratégica para asegurar la estabilidad del entregable.
</li>
</ul>
<h3>2.3. Especificaciones del Proyecto</h3>
<p>Se desarrollaron las pantallas clave solicitadas:</p>
<ul>
<li>
<strong>Pantallas de Acceso:</strong> Splash Screen, Login Page y Signup Page.
</li>
<li>
<strong>Gestión de Usuario:</strong> Home Page (estado activo y vacío) y Profile Page.
</li>
<li>
<strong>Operaciones Financieras:</strong> Interfaces para Enviar y Solicitar Dinero.
</li>
</ul>

<h2>3. Disclaimers y Consideraciones Finales</h2>
<ul>
<li>
<strong>Validación del Código por Hardware:</strong> Debido a limitaciones técnicas de hardware que impidieron el uso de un emulador, el desarrollo se realizó basándose estrictamente en la teoría del código y <strong>sin la previsualización del editor de Android Studio</strong>. El código es sintácticamente correcto y sigue la lógica de programación requerida, aunque no pudo verificarse en un entorno de ejecución real.
</li>
<li>
<strong>Uso de Metodología Agile:</strong> La gestión de este proyecto se centró en la entrega de un Producto Mínimo Viable (MVP) funcional y visualmente pulido.
</li>
<li>
<strong>Oportunidades de Mejora:</strong> Se reconoce la posibilidad de optimizar el uso de márgenes y paddings en una siguiente fase de desarrollo para alcanzar una precisión técnica total, además de la futura implementación de tipografías personalizadas.
</li>
</ul>
